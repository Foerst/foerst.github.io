第一章

1.输出print()函数print(“xxxx(变量)”)

2.常量let 变量var

3.type annotation 类型注解 （var index: Int= 12）

4.comment:注释 cmd+/ 或 /………/

5.可以不写分号；

6.整形 Int, UInt, Int8, UInt8, Int16, UInt16,

var minInt = Int.min

7.浮点型Float 精确度小数点后6位，Double 15位，申明浮点数时默认是double类型

8.类型安全和类型推断 type safe & type inference 根据 赋值的literal value(字面量)进行类型推断

let 十进制的10 = 10 （无前缀，直接写）

let 10亿 = 1_000_000_000

14.optional (可选值)

forced unwrapping:强制拆包/解析 可选值后添加！可以访问包装的存在值（若不存在回报运行时错误）

optional binding可选绑定 可以判断可选类型是否包含值，如果包含就把值赋给一个临时常量或者变量。可选绑定可以用在if和while语句中来对可选类型的值进行判断并把值赋给一个常量或者变量。可选绑定的模式为：

if let actualNumber = Int(possibleNumber) {
    //do something here
}

可以翻译为：“If the optional Int returned by Int(possibleNumber) contains a value, set a new constant called actualNumber to the value contained in the optional。If the conversion is successful, the actualNumber constant becomes available for use within the first branch of the if statement. It has already been initialized with the value contained within the optional, and so there is no need to use the ! suffix to access its value.”

隐式解析可选类型(implicitly unwrapped optionals)。把想要用作可选的类型的后面的问号(String?)改成感叹号(String!)来声明一个隐式解析可选类型。这是语法糖，避免在已确定可选值包含值的情况下不必要的可选绑定。（IB中控件就是这样，连线之后确定一定有值）

15.nil

即无值状态 通过对可选类型变量赋值使其成为无值状态。“nil cannot be used with nonoptional constants and variables. ” 定义无初始值的可选类型变量时默认为nil.

16.异常处理（error handling）

形式如下:

函数声明时添加throws关键字
func canThrowAnError() throws {
// this function may or may not throw an error
}
调用时 do包裹，catch处理
do {
try canThrowAnError()
// no error was thrown
} catch Error.NotFoundFile {
// an error was thrown
}catch Error.Unknown {
//deal matched error
}

17.Assertions(断言)

原理：条件成立继续运行，不成立，代码结束运行，app终止。assert(condition, msg) condition == false时，输出msg.msg可以省略。
作用：调试输出（注意：release模式下不可用，debug下可以调试输出）

第二章 基本操作符

基本操作符分为一元（unary）、二元(binary)和三元(ternary)运算符。

1.Assignment Operator(赋值运算符)

let b = 10 //b赋值为10

2.Arithmetic Operators（算术运算符）

+，-，*，/ ，注意的是+可操作字符串，值相加默认不允许值溢出（overflow）

3.取余（Remainder Operator）

注意，取余的2个操作数均可为浮点数。计算公式为：

a = (b x some multiplier) + remainder
========> a%b = remainder

比如 -9 % 4 = -1， 因为“-9 = (4 x -2) + -1”

8 % 2.5 = 0.5，因为 “8 = (2.5 x 3) + 0.5”

4.Increment and Decrement Operators(自增 自减 运算符)

++、 – 返回值跟c一样,比如: var a = 10 var b = ++a(++a有返回值，即加1之后的a)

5.Unary Minus Operator&Unary Plus Operator（取负&取正）

-a +b…

6.Compound Assignment Operators(组合赋值)

比如： a += 2 即时：a = a + 2

注意：这样写“let b = a += 2”是错误的，因为组合赋值没有返回值，这跟自增或自减是不一样的。

7.Comparison Operators（比较操作符）

== 、 ！= 、 > 、< 、<=、 >= 返回Bool类型

8.Ternary Conditional Operator（三目条件操作符）

syntax: question ? answer1 : answer2

If question is true, it evaluates answer1 and returns its value; otherwise, it evaluates answer2 and returns its value.

9.Nil Coalescing Operator(a ?? b) 空合并操作符

“The nil coalescing operator (a ?? b) unwraps an optional a if it contains a value, or returns a default value b if a is nil. The expression a is always of an optional type. The expression b must match the type that is stored inside a.

The nil coalescing operator is shorthand for the code below:

a != nil ? a! : b

例子：

let defaultColorName = "red"
var userDefinedColorName: String?   // defaults to nil
var colorNameToUse = userDefinedColorName ?? defaultColorName
// userDefinedColorName is nil, so colorNameToUse is set to the default of "red”

10.Range Operators（范围操作符）

  • Closed Range Operator（闭区间）
    a…b 即[a,b]
  • Half-Open Range Operator（半开区间）
    a..<b 即[a,b)

11.Logical Operators（逻辑操作符）

用来操作或组合bool值或表达式。

Logical NOT (!a)

Logical AND (a && b)：a和b同时为true时返回true,有短路原则

Logical OR (a || b)：有任意一个是true即返回true

注意：表达式可任意组合，添加“（）”可增强阅读性。

3.Conditional Statements(条件语句)

  • if

    条件不需要括号括起来,条件必须能够返回Bool值。

  • switch

    switch some value to consider {
case value 1:
    respond to value 1
case value 2,
value 3:
    respond to value 2 or 3
default:
    otherwise, do something else
}

    注意：

    case和default必须包含所有需要考虑的值，如果case已经包含了所有可能的值，default分支可以省略。

    特点：

    a.没有隐含的贯穿（No Implicit Fallthrough）:执行的case语句后没有break,case分支执行完后整个switch语句就执行完毕。

    b.case分支必须包含可执行的语句，否则报错。

    c.多个case值可用“，”分开。

    d.可以间隔匹配（Interval Matching）

    case 1..<5:
    naturalCount = "a few”

    f.case值可用元组

    let somePoint = (1, 1)
switch somePoint {
case (0, 0):
    print("(0, 0) is at the origin")
case (_, 0):
    print("(\(somePoint.0), 0) is on the x-axis")
default:
    print("(\(somePoint.0), \(somePoint.1)) is outside of the box")
}

    g.可用值绑定（Value Bindings）（A switch case can bind the value or values it matches to temporary constants or variables, for use in the body of the case. This is known as value binding, because the values are “bound” to temporary constants or variables within the case’s body.）

    let anotherPoint = (2, 0)
switch anotherPoint {
case (let x, 0):
    print("on the x-axis with an x value of \(x)")

    h.case分支可以使用where分句匹配额外的条件（dynamic filter）

    let yetAnotherPoint = (1, -1)
switch yetAnotherPoint {
case let (x, y) where x == y:
    print("(\(x), \(y)) is on the line x == y")

4.Control Transfer Statements（控制转移语句）

Swift有5种控制转移语句:

  • continue：结束当前循环，进入下一次循环
  • break：跳出整个循环

  • fallthrough：case分支后添加fallthrough关键字（c中break的位置）,可以使之后的语句继续执行，而不退出switch语句。

    “let integerToDescribe = 5
var description = "The number \(integerToDescribe) is"
switch integerToDescribe {
case 2, 3, 5, 7, 11, 13, 17, 19:
    description += " a prime number, and also"
    fallthrough
default:
    description += " an integer."
}
print(description)
// prints "The number 5 is a prime number, and also an integer.”
注意：The fallthrough keyword does not check the case conditions for the switch case that it causes execution to fall into. The fallthrough keyword simply causes code execution to move directly to the statements inside the next case (or default case) block, as in C’s standard switch statement behavior.
翻译：fallthrough关键字总是跳到下一个case（或default）语句执行，从不检查跳入的case语句的条件。（？？）

  • return:函数返回

  • throw：抛出异常

5.Labeled Statements（标签语句）

给循环或条件语句起个状态标签，然后用break/continue进行跳转执行（类似c中goto），break跳出，continue结束当前循环，进入下一次循环。
语法：

label name: while condition {
statements
}

6.Early Exit

guard关键字：跟if语句类似，但目的是增强代码可读性，并且可选绑定的变量或常量作用域跟if语句不同。
例子：

func greet(person: [String: String]) {
    guard let name = person["name"] else {
        return
    }
    print("Hello \(name)!")
    guard let location = person["location"] else {
        print("I hope the weather is nice near you.")
        return
    }

    print("I hope the weather is nice in \(location).")
}
greet(["name": "John"])
// prints "Hello John!

7.Checking API Availability

if #available(iOS 9, OSX 10.10, *) {
    // Use iOS 9 APIs on iOS, and use OS X v10.10 APIs on OS X
} else {
    // Fall back to earlier iOS and OS X APIs
}
The last argument, *, is required and specifies that on any other platform

第六章 函数（Function）

即：执行特定任务的代码块。

1.Defining and Calling Functions

func关键字开头 函数名，参数名（后跟类型），–> 后跟返回值。调用 函数名（参数）

2.Function Parameter Names:

  • external parameter name:主要是函数调用时参数含义辅助功能。默认情况下，第一个参数的外部参数名省略，其余的使用local parameter name作为外部参数名。当然，你也可以在函数实现时指定外部参数名，格式如下：

    func someFunction(externalParameterName localParameterName: Int){
    // function body goes here, and can use localParameterName
    // to refer to the argument value for that parameter
}
//在本地参数名前写外部参数名，之间有空格

  • local parameter name:函数实现时用到的局部参数名。

3.Omitting External Parameter Names—忽略外部参数名

write an underscore () instead of an explicit external name for that parameter——用下划线“”替代外部参数名，这样调用的时候就可以不写外部参数名了。

4.Default Parameter Values—-默认参数值

参数类型后加默认值即可

func someFunction(parameterWithDefault: Int = 12) {
    // function body goes here
    // if no arguments are passed to the function call,
    // value of parameterWithDefault is 12
}

5.可变参数（Variadic Parameters）

变参可接受0或多个特定类型的值。在参数类型后面插入(…)表示可变参数。可变参数的参数名是特定类型的数组类型，在函数体中作为数组使用。A function may have at most one variadic parameter.

func arithmeticMean(numbers: Double...) -> Double {
    var total: Double = 0
    for number in numbers {
        total += number
    }
    return total / Double(numbers.count)
}
arithmeticMean(1, 2, 3, 4, 5)”
// returns 3.0
